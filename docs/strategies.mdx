---
title: 'Routing Strategies'
description: 'Choose how your API traffic is distributed across keys'
---

PoolSwitch supports multiple routing strategies to determine which API key should be used for a given request. Choosing the right strategy is crucial for optimizing throughput, avoiding rate limits, and minimizing costs.

## `round_robin`

The simplest strategy. It evenly rotates requests across all available, healthy keys in order.

**Best for**: General load balancing when all keys have similar quotas and rate limits.

## `least_used`

Selects the key that has processed the lowest total number of requests so far.

**Best for**: Balancing overall load accurately over a long period. Useful when you add a fresh key to the pool and want the system to aggressively utilize it until its usage catches up with older keys.

## `random`

Selects a random healthy key from the pool for each request.

**Best for**: High concurrency environments where deterministic selection algorithms might suffer from race conditions or thundering herd problems.

## `quota_failover`

The most advanced strategy. It evaluates several factors to pick the optimal key:

1. **Estimated Remaining Quota**: Favors keys with a higher remaining budget (`monthly_quota` minus estimated usage).
2. **Rate Limits**: Disfavors keys that have received recent 429 Too Many Requests responses.
3. **Failover**: Falls back gracefully to other healthy keys if the preferred key enters a cooldown state.

**Best for**: Production environments relying heavily on varied quota tiers (e.g., mixing free tier keys with paid tier keys).

## Key States

Keys can transition between several states regardless of the strategy used:

- **Healthy**: Ready to serve traffic.
- **Cooldown**: If an upstream provider returns a `429 Too Many Requests` or quota exhaustion error, PoolSwitch places the key into a temporary timeout (configured by `cooldown_seconds`). The key is ignored by routing strategies until the cooldown expires.

By automatically applying cooldowns and seamlessly picking the next best key, PoolSwitch ensures high availability for your upstream API calls without manual intervention.
